# Easytrip: Travel Itinerary Planner

Easytrip is a Django-based web application designed to help users quickly generate and manage travel itineraries based on their preferences.

## System Architecture

The project is built using the Django framework (Python) and utilizes a SQLite database by default (or PostgreSQL/others depending on deployment configuration). It consists of a primary app called `planner` which handles the core functionality, including trip generation, user dashboard, and authentication.

### Core Components

1.  **Authentication System**:
    *   Utilizes Django's built-in `User` model for handling user accounts.
    *   Views for **Signup**, **Login**, and **Logout** (`signup_view`, `login_view`, `logout_view`).
    *   Unauthenticated users can generate trips, but authenticated users can save and view their trips in a personalized dashboard.

2.  **Trip Generation & APIs** (`planner/views.py - home`):
    *   Users submit a form with details such as **destination**, **trip length**, **group size**, **start/end dates**, and **interests**.
    *   **Wikipedia API Integration**: Fetches an overview or summary of the chosen destination to provide users with initial background information.
    *   **Nominatim (OpenStreetMap) API Integration**: Geocodes the destination to retrieve precise latitude and longitude coordinates.
    *   **LoremFlickr Integration**: Dynamically fetches a background image relevant to the generated city/destination query.
    *   After fetching data, the system creates a `Trip` record and automatically generates placeholder `ItineraryDay` records based on the requested trip length.

3.  **Data Models** (`planner/models.py`):
    *   `Trip`: The central model representing a generated itinerary. It contains the destination name, dates, group size, interests, as well as API-fetched data like the Wikipedia overview, image URL, and geographical coordinates. It optionally links to a `User` via a foreign key.
    *   `ItineraryDay`: Represents a single day within a `Trip`. Linked to a `Trip` via a foreign key. Contains a day number and a description.

4.  **Views & Routing**:
    *   **Home (`/`)**: Landing page with the trip generation form. Also displays a grid of recent trips for logged-in users.
    *   **Dashboard (`/dashboard`)**: A user-specific page listing all trips generated by the currently logged-in user, ordered by creation date.
    *   **Trip Detail (`/trip/<id>`)**: Displays the full details of a specific trip, along with its generated itinerary days.
    *   **Delete Trip (`/delete_trip/<id>`)**: Endpoint to remove a saved trip from the user's dashboard.

## External Services & Integrations

*   **Wikipedia REST API**: Used to fetch quick summaries for locations (`https://en.wikipedia.org/api/rest_v1/page/summary/...`).
*   **OpenStreetMap Nominatim API**: Used for geocoding to retrieve `lat` and `lon` for mapping or location context (`https://nominatim.openstreetmap.org/search...`).
*   **LoremFlickr**: Used for dynamic destination imagery (`https://loremflickr.com/...`).

## Project Structure

*   `easytrip/`: The core Django project folder containing settings, core URLs, and WSGI/ASGI configurations.
*   `planner/`: The main application containing the models, views, and specific URL routing for the travel planner.
*   `templates/`: Contains HTML files corresponding to the views (e.g., `home.html`, `dashboard.html`, `detail.html`, `login.html`, `signup.html`).
*   `static/`: Directory for static assets such as CSS styles and images.
